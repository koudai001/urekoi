# コンポーネント構造の規定

## 背景
- 現状`components/ui/`(shadcn由来の汎用UI)と`components/`直下(機能コンポーネント)の2階層が自然にできている
- atomic design(atoms/molecules/organisms)ほど厳密にはせず、今の2階層をルールとして明文化したい
- Storybookでストーリーを書く際の置き場所も合わせて決めたい

## ルール(案)
- `components/ui/`: 汎用UIプリミティブ。状態を持たず、propsで振る舞いが決まるもの(ボタン・入力欄など)。ビジネスロジック・データ取得は持たない
- `components/`直下: 機能コンポーネント。`ui/`のプリミティブを組み合わせて作る。データ取得・状態管理を持ってよい
- サブフォルダ分けはしない(ドメイン単位のサブフォルダは今の規模では不要)
- ストーリーファイルは対象コンポーネントと同じディレクトリに置く(例: `components/ui/button.tsx` → `components/ui/button.stories.tsx`)
